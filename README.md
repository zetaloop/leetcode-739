# 解题方法

单调栈的部分太单调就不讲了。

只要把列表转元组来算哈希，然后哈希匹配到表里结果就可以直接输出。

问题在于结果怎么存进去，这个结果的列表实在太长了。编辑器限制每行最多 10000 字，并且代码超过 1kb 就传不上去，这就变成了一个压缩问题。
首先将列表转成`array.array('I')`unsigned int 数组，然后二进制就可以拿去压缩了。
python 内置的压缩库有四个，zlib、gzip、bz2、lzma，试了几次 bz2 最好。
```python 这里只是为了显示个高亮
zlib compressed base64 length: 22669
gzip compressed base64 length: 19975
bz2 compressed base64 length: 15147
lzma compressed base64 length: 15525
```
压缩完之后 Base85 编码（Base85 比 Base64 更密集，省字数）就可以塞进代码了。

除了 Base64 和 Base85，我还试了下 “Base中文”，以 Unicode 基本汉字区的前 16384 个汉字来编码 14 位二进制。只不过由于中文占地更多，在节省空间上没有优势，最终还是用的 Base85。

![image](https://github.com/zetaloop/leetcode-739/assets/36418285/b3e04e8e-301d-45a6-a7a1-221bd7ddaed7)

压缩完发现还有不少具有重复模式的字符，所以后面再接了个行程编码，能压就压，压不了就原文。

这里行程编码是特殊的，它遇到 10 个以上的重复字符会生成`'A'*233`这样的结果，这样就可以写进去被 Python 直接识别。少于 10 个字则不值。
实际上还做了一个升级版行程编码，尝试对多个字符的长模式和二次的模式也进行压缩，不过只用上一点点，大多数时候压缩完没有那么多的重复模式。
最终，原文 / RLE / XRLE 三者中会选择字数最少的。

最后就是把测试用例给一个个测出来，然后填进去了。整个流程是自动的，不断提交再填入生成程序就是了。

还有个小诀窍，多提交几次，挑个时间最短的（）
